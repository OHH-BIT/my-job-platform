# 正则表达式错误修复总结

## 🐛 问题描述

**错误类型**：`Runtime SyntaxError: Invalid regular expression: Nothing to repeat`

**错误原因**：在构建技能关键词的正则表达式时，直接使用了包含特殊字符的技能名（例如 `C++`）。在正则语法中，`+` 是量词，表示"重复前一个字符"，单独出现在开头或连续出现会导致语法错误。

**受影响文件**：
1. `src/lib/resume-analyzer.ts` - 第333行和第171行
2. `src/lib/resume-checker.ts` - 第48行（修复后第60行）

---

## ✅ 修复方案

### 1. 创建工具函数 `escapeRegExp`

**位置**：
- `src/lib/resume-analyzer.ts` - 第22-28行
- `src/lib/resume-checker.ts` - 第22-28行

**功能**：自动将字符串中的正则特殊字符进行转义

**代码**：
```typescript
/**
 * 转义正则表达式中的特殊字符
 * @param str - 需要转义的字符串
 * @returns 转义后的字符串，可直接用于正则表达式
 */
function escapeRegExp(str: string): string {
  // 需要转义的特殊字符：\ . * + ? ^ $ { } ( ) [ ] | /
  return str.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
}
```

**转义规则**：
- `.` → `\.`
- `*` → `\*`
- `+` → `\+`
- `?` → `\?`
- `^` → `\^`
- `$` → `\$`
- `{` → `\{`
- `}` → `\}`
- `(` → `\(`
- `)` → `\)`
- `[` → `\[`
- `]` → `\]`
- `|` → `\|`
- `\` → `\\`

## 💡 最佳实践

### 1. 永远不要直接将用户输入用于正则表达式

**❌ 错误做法**：
```typescript
const userInput = 'C++';
const regex = new RegExp(userInput, 'i');  // 危险！
```

**✅ 正确做法**：
```typescript
const userInput = 'C++';
const escapedInput = escapeRegExp(userInput);
const regex = new RegExp(escapedInput, 'i');  // 安全
```

### 特殊字符列表
- `.` - 任意字符（除换行符）
- `*` - 0次或多次
- `+` - 1次或多次
- `?` - 0次或1次
- `^` - 字符串开头
- `$` - 字符串结尾
- `{` `}` - 重复次数
- `(` `)` - 捕获组
- `[` `]` - 字符集
- `|` - 或
- `\` - 转义符
